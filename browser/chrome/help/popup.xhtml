팝업 제어 하기

이 문서는 &brandFullName;에서 팝업 제어를 위해 사용하는 모든 &pref.plural; 에 대하여 설명합니다.

세부 항목:
  • 팝업이란?
  • 팝업 차단 &pref.pluralCaps;

팝업이란?

팝업 창 또는 팝업은 사용자의 허락없이 자동적으로 나타나는 창을 말합니다. 이 창은 크기가 다양하지만 보통 전체 화면을 덮을 정도로 크지는 않습니다. 현재 &brandShortName; 창 위에 열리는 것도 있고, &brandShortName; 창 아래에 열리는 것(팝언더)도 있습니다.

&brandShortName;는 사용자가 팝업 및 팝언더를 도구 > &pref.pluralCaps; 메뉴의 내용 설정에서 팝업 창 차단을 통해 관리할 수 있도록 합니다. 팝업 차단은 기본적으로 켜져있으므로, &brandShortName;에서 팝업을 차단하기 위해 이 기능 켤 것을 걱정을 할 필요는 없습니다.

팝업이 차단되고 있으면, &brandShortName;는 정보 표시줄을 띄우고, 상태 표시줄에 아이콘을 표시합니다. 정보 표시줄에서 &pref.pluralCaps;이나 상태 표시줄에서 아이콘을 클릭하면, 메뉴에 다음과 같은 선택 사항이 표시됩니다:

  • 팝업 창 허가
  • 팝업 창 차단 &pref.pluralCaps;…
  • 팝업 창이 차단될 때 메시지를 표시하지 않음
  • (차단된 팝업 보이기)

팝업 차단 기능이 일부 웹 사이트와는 충돌할 수도 있습니다: 은행 사이트를 비롯한 일부 웹 사이트는 중요한 기능을 수행하기 위해서 팝업을 사용합니다. 모든 팝업을 차단하면 이러한 기능이 정지됩니다. 특정 웹 사이트에 대해서는 팝업을 허용하면서 그 외 사이트는 차단하기를 원한다면, 해당 사이트를 허용 사이트 목록에 추가할 수 있습니다.

팝업 차단 기능이 항상 작동하는 것은 아닙니다: 비록 &brandShortName;가 대부분의 팝업을 차단하기는 하지만, 일부 사이트는 아직 알려지지 않은 방법을 사용하여, 팝업이 차단되어 있음에도 불구하고, 팝업을 띄울 수도 있습니다.

팝어 차단 &pref.pluralCaps;

팝업 차단 &pref.pluralCaps; 관리자는 &pref.menuPath; 메뉴의 내용 설정에 있습니다.

거기서 다음과 같은 것을 설정할 수 있습니다:

  • 팝업 창 차단: 이 &pref.singular;이 선택 해제되어 있으면 팝업 차단 기능이 정지됩니다.
  • 예외 목록 대화 상자: 팝업을 허용하기를 원하는 사이트 목록입니다. 이 대화 상자에서는 다음을 수행할 수 있습니다:
    • 허가: 클릭하면 웹 사이트가 예외 목록에 추가됩니다.
    • 제거: 클릭하면 웹 사이트가 예외 목록에서 삭제됩니다.
    • 모두 제거: 클릭하면 전체 예외 목록 리스트가 삭제됩니다.

주의: 팝업 차단 기능은 항상 작동하는 것이 아니며 일부 웹 사이트와는 충돌할 수도 있습니다. 팝업 차단에 관한 더 자세한 정보는 팝업이란를 참조합니다.
