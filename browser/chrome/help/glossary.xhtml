이 용어 사전은 사용자 참조용으로만 제공됩니다; 아래 정의된 용어나 개인 정보에 대한 설명과 언급된 기술에 대한 보안 세부 정보를 완벽하거나 공신력있는 기술서로 신뢰할는 수 없다는 뜻입니다.

용어 사전

인증 (authentication)
비밀번호, 인증서, 개인 식별 번호(PIN), 또는 기타 정보를 이용하여 컴퓨터 네트워크 상에서 신원을 확인하는 것을 말합니다.
북마크 (bookmark)
저장되어 있는 페이지 주소(URL)로 북마크 도구 모음에 있는 북마크 아이콘을 클릭하거나 북마크 메뉴에서 북마크 이름을 클릭하여 해당 페이지로 쉽게 이동할 수 있습니다.
북마크 도구 모음 (Bookmarks Toolbar)
사용자가 설정할 수 있는 도구 모음으로 &brandShortName;에서는 기본적으로 주소 표시줄 바로 아래에 나타납니다. 이 도구 모음에는 즐겨찾는 북마크 (또는 북마크 그룹을 담고 있는 폴더) 버튼이 있으며 사용자가 추가하거나 삭제할 수 있습니다.
캐시 (cache)
수집된 웹 페이지 복사본으로 사용자 컴퓨터의 하드 디스크나 메모리(RAM)에 저장됩니다. &brandShortName;는 웹을 탐색하는 동안 이런 웹 페이지 복사본을 수집합니다. 링크를 클릭하거나 URL을 입력하여 특정 웹 페이지를 불러올 때, 캐시에 이미 복사본이 있으면, &brandShortName;는 캐시된 복사본과 원본을 비교합니다. 아무런 차이가 없으면, &brandShortName;는 원본을 다시 불러오기 보다는 캐시된 복사본을 이용함으로써 다운로드하고 처리하는데 걸리는 시간을 절약합니다.
인증서 (certificate)
신분증(ID card)에 해당하는 디지털 도구입니다. 인증서는 개인의 이름, 회사나 기타 실체을 명시하고 있으며 인증서에 포함된 공개키가 이 실체에 속한다는 것을 보증합니다. 메시지나 기타 자료에 전자적으로 서명 하면, 이 메시지에 대한 디지털 서명은 비밀키를 이용하여 생성되는데, 비밀키는 사용자의 인증서에 있는 공개키에 대응되는 인증키입니다.
클라이언트 (client)
(인터넷 브라우저와 같은) 소프트웨어로, 보통 다른 컴퓨터에서 운영되고 있는 서버에 요청을 보내고 정보를 받아오는 소프트웨어입니다. 클라이언트 소프트웨어가 실행되고 있는 컴퓨터를 또한 클라인트라고 합니다.
쿠키 (cookie)
일부 웹 사이트가 사용자의 컴퓨터에 저장해 놓은 작은 용량의 정보입니다. 웹 사이트를 방문하면, 이런 웹 사이트는 &brandShortName;에게 쿠키를 사용자의 컴퓨터에 저장할 수 있는지를 물어옵니다. 나중에, 이 사이트를 재방문하면, &brandShortName;는 이 사이트에 속한 쿠키를 되돌려 보냅니다. 쿠키는 웹 사이트가 사용자의 정보를, 마치 쇼핑 카트의 내용물과 같이, 보관하는 것을 도와 줍니다. 쿠키 옵션을 설정하여 쿠키가 어떻게 사용되고 얼마나 많은 정보를 웹 사이트가 저장하도록 허용할 것인지를 제어할 수 있습니다.
암호화기법 (cryptography)
정보를 뒤섞어 알아보지 못하게 하거나(암호화) 다시 알아볼 수 있게 해독하는(복호화) 기법이나 방법입니다. 예로, 암호화 기술은 상거래 웹 사이트와 &brandShortName; 간에 전달되는 정보를 암호화하고 해독하는데 이용됩니다.
복호화 (decryption)
암호화된 자료를 해독하는 조작을 말합니다. 더불어 암호화도 참조합니다.
디지털 (digital) ID
인증서를 참조합니다.
암호화 (encryption)
정보를 뒤섞어서 그 의미를 감추는 조작을 말합니다. 예를 들어, 컴퓨터 간에 접속이 암호화 되어 있으면, 제삼자가 이 접속을 통해 전달되는 정보를 해독하거나 복호화하기 매우 어렵게 됩니다. 암호화된 정보는 적절한 해독 키를 가진 사람만이 복호화 할 수 있습니다.
XML (Extensible Markup Language)
정보를 서술하는 오픈 표준 규격입니다. HTML과는 달리, XML에서는 웹 페이지 개발자 고유의 테그를 정의할 수 있습니다. 더 자세한 정보는 온라인 W3C 문서 XML (Extensible Markup Language)를 참조합니다.
RSS 피드 (feed)
다른 웹 사이트에 대한 링크 목록을 있는 XML 웹 페이지입니다. 특수한 프로그램이 이런 피드 페이지를 읽어 링크에서 표제 목록을 생성하며 자동적으로 목록 변경 사항을 업데이트합니다. 뉴스 웹 사이트는 RSS 피드 페이지를 이용하여 신속하게 최신 표제를 게시하고, 개인 온라인 잡지는 보통 피드 페이지를 이용하여 신속하게 방문자들에게 뉴스 목차를 알려줍니다. 더불어 라이브 북마크도 참조합니다.
FTP (File Transfer Protocol)
네트워크를 통해 파일을 한 컴퓨터에서 다른 컴퓨터로 전송하는 표준 규격입니다. &brandShortName;는 FTP를 이용하여 파일을 받을 수 있습니다.
FIPS PUBS 140-1
연방 정보 처리 표준 안(Federal Information Processing Standards Publications) 140-1은 암호화 모듈 — 자료를 암호화하고 복호화하거나 (디지털 서명을 작성하거나 확인하는 것과 같은) 기타 암호화 조작을 수행하는 하드웨어나 소프트웨어 — 구현에 대한 미국 정부의 표준 규격입니다. 미국 정부에 공급되는 많은 제품이 FIPS 표준을 준수해야 합니다.
홈 페이지 (home page)
&brandShortName;가 실행되거나 홈 단추를 클릭할 때마다 표시되는 웹 페이지입니다. 또한 사이트의 메인 페이지를 가리키는 말로도 사용되며, 이 페이지를 통해 해당 사이트의 나머지 페이지를 탐색할 수 있습니다.
HTML (Hypertext Markup Language)
웹 페이지에서 사용하는 문서 형식입니다. HTML 표준 규격은 테그나 코드를 정의하고 있으며 이들은 텍스트 레이아웃, 글꼴, 스타일, 그림 및 웹 페이지를 꾸미는데 사용하는 기타 구성 요소를 정의하는데 사용됩니다.
인터넷 (Internet)
수많은 컴퓨터를 연결하는 전세계적인 네트워크로 TCP/IP와 같은 표준 통신 규약을 사용하여 다른 컴퓨터와 통신을 합니다. 원래는 미국 군사용으로 1969년에 개발되었지만, 점점 성장하여 교육 연구 기관을, 그리고 1990년대 후반에는, 무수히 많은 기업과 기관과 개인을 포함하게 되었습니다. 오늘날 인터넷은 전자 메일, World Wide Web 탐색, 메신저, 메일리 리스트와 뉴스그룹, 그리고 많은 기타 다른 목적으로 사용됩니다.
IP 주소 (Internet protocol address)
TCP/IP 네트워크에서 컴퓨터이 주소입니다. 모든 컴퓨터에는 인터넷에서 하나의 IP 주소가 할당 되어 있습니다. 클라이언트는 영구 IP 주소를 가지고 있을 수도 있고, 네트워크에 접속할 때마다 유동적으로 할당되는 주소를 가지고 있을 수도 있습니다.
자바 (Java)
Sun Microsystems사가 개발한 프로그램 언어입니다. 단일 자바 프로그램은 다양한 기종의 컴퓨터에서 실행될 수 있으며, 따라서 프로그래머는 기종마다 각기 다른 독립적인 포로그램을 만들 필요가 없습니다. 자바가 설치되고 나면, brandShortName;는 자동적으로 자바 프로그램(애플릿이라고도 함) 을 다운로드하여 실행할 수 있습니다.
자바스크립트 (JavaScript)
웹 사이트 제작에 흔히 사용되는 스크립트 언어입니다. 프로그래머는 자바스크립트를 이용하여 더욱 상호작용하는 웹 페이지를 만들수 있습니다; 자바스크립트는 보통 폼과 선택 단추의 동적으로 유효성 검사하는데 사용됩니다. 자바스크립트는 자바와 함께 사용될 수도 있지만, 기술적으로 별개의 언어입니다. 즉, 자바가 있어야만 자바스크립트가 정상적으로 작동하는 것이 아닙니다.
라이브 북마크 (Live Bookmark)
특수한 형태의 북마크로 RSS 피드에 있는 링크를 모아둔 폴더처럼 동작합니다. 라이브 북마크를 생성하려면 피드를 제공하는 사이트를 방문하여 주소 표시줄에서 현재 페이지 구독하기... 단추를 클릭하여 피드 페이지를 연 다음, 페이지 첫 부분에서 RSS 구독기 선택 단추를 클릭하여 대화 상자를 엽니다. 대화 상자에서 라이브 북마크 사용하기를 선택하고, 지금 구독 단추를 클릭하여 라이브 북마크에 추가 대화 상자를 열어 북마크 이름을 입력하여 저장될 위치를 선택합니다.
주소 표시줄 (Location Bar)
&brandShortName; 창 상단에 있는 입력 란(또한 수반된 단추 아이콘)으로 URL이나 검색어를 입력할 수 있습니다.
기본 암호 (master password)
저장된 암호화와 기타 개인 정보를 보호하는데 사용하는 암호입니다. 이런 정보에 접근하려고 하면 &brandShortName;가 기본 암호를 물어올 것입니다. 다중 보안 장치를 가지고 있으면, 각 보안 장치마다 독립된 기본 암호를 요구할 것입니다.
표준 도구 모음 (Navigation Toolbar)
&brandShortName; 창 상단에 있는 도구 모음으로 뒤로와 앞으로와 같은 단추를 담고 있습니다.
암호 관리자 (Password Manager)
&brandShortName;의 한 기능으로 사용자의 이름과 암호의 일부나 전부를 하드 디스크에 저장하고 해당 사이트를 방문할 때 자동적으로 입력시켜 줍니다.
피싱 (Phishing)
피싱은 신원을 도용하는 한 형태로 악의적인 웹 사이트가 진짜 사이트를 가장하여 비밀번호, 계좌정보, 또는 신용카드 번호와 같은 민감한 정보를 빼내는 것을 말합니다. 피싱 공격은 보통 전자메일 메시지 형태로 오며, 정말로 진짜처럼 보이지만 위조된 사이트에서 사용자가 자신의 개인 정보를 업데이트하도록 유도합니다. 대개 메인 창에 표시되는 사이트는 '진짜'이지만, 팝업 페이지는 '가짜'입니다. 진짜를 보고 안심한 사용자가 팝업에 표시된 입력란에 인증번호나 비밀번호, 신용카드번호 등의 비밀을 입력·송신하면 피싱을 하려는 자에게 정보가 송신됩니다. &brandShortName;는 피싱으로부터 사용자를 보호하기 위하여 피싱 검사 기능을 제공합니다.

공개키 암호화 표준 규격으로 스마트 카드 같은 보안 장치를 관장합니다.
플러그인 (Plugin)
플러그인은 &brandShortName;에 음성이나 영상 자료의 재생 능력과 같은 새로운 기능을 부여합니다. 다른 보조 응용프로그램과는 달리, 플러그인은 자기 자신을 메인 &brandShortName; 설치 디렉토리 내에 있는 플러그인 디렉토리에 설치하고 일반적으로 &brandShortName; 내에서만 (내부적으로) 열릴 수 있습니다. 예를 들어, 음성 플러그인은 사용자가 웹 페이지나 전자 메일 메시지에 있는 음성 파일을 들을 수 있도록 해 줍니다. 매크로미디어 플래시 플레이어(Macromedia Flash Player)와 자바(Java)는 둘다 플러그인 응용프로그램의 좋은 예입니다.
비밀키 (private key)
공개키 암호화기법에 사용되는 한 쌍의 키중 하나입니다. 비밀키는 안전하게 보관이되며 상응하는 공개키로 암호화된 자료를 복호화하는데 사용됩니다.
프록시 (proxy)
중간 또는 "중개" 프로그램으로 서버와 클라이언트 양쪽 모두의 역할을 하며, 다른 클라이언트 대신 요청을 수행하고 결과를 전달하기 위한 목적으로 사용됩니다.
검색 엔진/사이트 (search engine)
웹 기반 프로그램으로 사용자가 특정한 정보를 WWW에서 검색하고 수집할 수 있도록 해 줍니다. 검색 엔진은 웹 문서 전체 텍스트를 검색할 수도 있고 키워드 목록에서 검색할 수도 있습니다; 검색 엔진은 또한 사서 로봇 프로그램을 이용하여 웹 문서를 수집하여 일일이 색인을 메기기도 합니다. 보통은 단어나 구문을 검색 창에 입력하면, 검색 엔진이 관련있는 웹 페이지에 대한 링크를 표시합니다.
보안 사이트 (secure site)
암호화 를 사용해서 &brandShortName;와 접속함으로써 악의적인 인터넷 사용자가 전송되는 자료를 훔쳐보지 못하도록하는 사이트입니다. 보안 사이트를 방문하면, &brandShortName;는 좌물쇠 아이콘을 상태 표시줄과 주소 표시줄에 표시합니다. &brandShortName;는 또한 (악의적 사이트가 사용자의 자료를 도용하지 못하도록) 사이트의 도메인명을 상태 표시줄에 보여주고 주소 표시줄의 배경을 노란색으로 바꿉니다.
일부 사이트는 보안과 비보안 자료 모두로 구성되어 있을 수 있습니다; 비보안 자료는 대개, 보증할 수는 없지만, 개인 정보가 아닙니다. 이런 사이트에 대해서, &brandShortName; 슬래시 표시가 있는 좌물쇠 아이콘을 상태 표시줄과 주소 표시줄에 표시하고 사이트의 도메인명을 상태 표시줄에 않아 현재 사이트가 완전한 보안 사이트는 아니라는 것을 알려줄 것입니다.
SSL (Secure Sockets Layer)
클라이언트와 서버 사이의 상호 인증을 통해 인증되고 암호화된 접속을 확립하기 위한 프로토콜입니다. SSL은 TCP/IP 위에서 그리고 HTTP, LDAP, IMAP, NNTP, 및 기타 상위 수준 네트워크 포로토클 아래에서 실행됩니다. TLS(Transport Layer Security)라고도 하는 새로운 IETF(Internet Engineering Task Force) 표준은 SSL을 기반으로 하고 있습니다. 더불어 인증과 암호화도 참조합니다.
서버 (server)
(웹 페이지를 서비스하는 소프트웨어와 같은) 소프트웨어로, 보통 다른 컴퓨터에서 실행되고 잇는 클라이언트로부터 요청을 받아 정보를 보내주는 소프트웨어입니다. 서버 소프트웨어가 실행되고 있는 컴퓨터를 또한 서버라고합니다.
상태 표시줄 (Status Bar)
&brandShortName; 창 하단에 나타나는 도구 모음입니다. 상태 표시줄 오른쪽 부분에 상태 아이콘이 나타납니다.
TLS
See Secure Sockets Layer (SSL).
TCP/IP (Transmission Control Protocol/Internet Protocol)
다양한 운영 체계를 실행하고 있는 컴퓨터들을 접속하는데 이용되는 유닉스 프로토콜입니다. TCP/IP는 필수적인 인터넷 프로토글이며 국제 표준이 되었습니다.
URL (Uniform Resource Locator)
&brandShortName;가 웹에서 어떻게 파일의 위치나 기타 리소스의 위치를 찾을 수 있는지를 알려주는 표준화된 주소입니다. (예, http://www.mozilla.org) URL을 &brandShortName; 주소 표시줄에 입력하여 웹 페이에 접급합니다. URL은 또한 한 웹 페이지에서 다른 웹 페이지로 이동하는 링크에도 사용됩니다. 인터넷 또는 웹 주소로도 알려져 있습니다.
웹 페이지 (web page)
WWW에 있는 단일 문서로 유일한 주소나 URL로 지정됩니다. 웹 페이지에는 텍스트, 하이퍼링크, 및 그림 등이 담겨 있습니다.
웹 사이트 (web site)
하이퍼링크로 연결된 여러 웹 페이지 그룹으로 회사, 단체, 또는 개인이 관리합니다. 웹 사이트에는 텍스트, 그림, 음성 및 영상 파일, 그러고 다른 웹 페이지에 대한 링크 등이 포함되어 있습니다.
WWW (World Wide Web)
웹이라고도 합니다. 인터넷의 한 부분으로 웹 서버에 의해 저장되고 (&brandShortName;와 같은) 클라이언트에 의해 보여지는 웹 페이지로 구성되어 있습니다.
