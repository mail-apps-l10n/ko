시작 페이지

&brandShortName;를 시작하면, 기본적으로 홈 페이지가 표시됩니다. 홈 페이지는 자주 방문하는 페이지로, 검색 사이트일 수도 있고, 또는 전자 메일 사이트일 수도 있으며, 이는 전적으로 사용자 선택에 달려있습니다. 이 섹션에서 홈 페이지를 바꿀 수도 있고 다른 페이지를 표시할 수도 있고 또는 여러 페이지를 지정할 수도 있습니다.

&brandShortName;를 시작할 때 표시할 페이지:
기본적으로 &brandShortName;는 시작하면서 홈 페이지 &pref.singular;에 대응하는 선택 상자 바로 아래에 있는 홈 페이지를 표시합니다. 또는 시작할 때 빈 페이지를 (아마도 인터넷에서 페이지를 불러오는데 필요한 시간을 줄이려고) 표시하려면 빈 페이지 &pref.singular;을 선택합니다.

또 다른 방법으로는 사용자가 인터넷 탐색을 마지막으로 멈춘 곳을 복원하여, 마지막으로 열었던 창과 탭을 다시 열게 할 수도 있습니다. 이는 운영 체계 업데이트를 설치하는 등으로 &brandShortName;를 닫게 될 때, 하고 있는 작업을 저장하는 유용한 방법이 될 수 있습니다. 이 &pref.singular;을 지정하려면 이전 종료시 창과탭를 선택합니다.

홈 페이지:

  • 기본 설정으로 복구를 클릭하면 홈 페이지가 기본 설정 홈 페이지로 초기화 됩니다.

탭 &pref.pluralCaps;

링크 여는 방법
이 &pref.singular;은 다른 응용 프로그램이나 웹 페이지 링크를 새 창으로 열 것인지 아니면 가장 최근에 열린 창의 새 탭으로 열 것이지를 제어할 수 있습니다.

주의: 링크를 새 탭에서 열기로 선택해 놓았다할지라도, 웹 페이지 제작자가 새 창이 특정 크기로 열리도록 지정하였다면, 새 창으로 열릴 것입니다. 왜냐하면, 이런 페이지는 정확히 특정 크기로 열려야만 하기 때문입니다.

다중 탭을 닫을 때 알려주기
다중 탭이 있는 창을 닫으려고 하면, &brandShortName;는 정말로 닫을 것인지를 물어올 것입니다. 이를 통해 실수로 현재 탭 대신 창 전체를 닫지 않도록 합니다. 이 선택 상자를 선택 해제하면 알림 창을 표시하지 않은 채 &brandShortName;가 자동적으로 창을 닫을 것입니다. 또한 많은 탭을 한 번에 닫으려고 할 때도, 정말로 닫을 것이지를 확인하게 합니다. 여러 탭을 한 번에 닫으려면, 탭 도구 모음에서 탭 막대 옆의 빈영역을 마우스 오른쪽 클릭하여 탭 모두 닫기를 선택하면, 현재 탭을 제외한 나머지 탭을 모두 닫습니다.

다중 탭을 열어서 &brandShortName;가 느려지것 같으면 알려주기
많은 탭을 열면 &brandShortName;가 느려질 수가 있습니다. 이 &pref.singular;을 선택하면 다중 탭으로 인해 &brandShortName;가 느려지는 것 같으면 알려줍니다.

탭 도구 모음을 항상 보이기
&brandShortName; 창에서 단 한 페이지만을 보고 있으면, 탭 도구 모음이 나타나지 않습니다. 하지만 이 &pref.singular;이 선택 되어 있으면 항상 탭 도구 모음이 표시될 것입니다.

새 탭에 링크를 열 때 바로 전환
웹 링크를 마우스 가운데 클릭(또는 &accelKey;를 누를 채 마우스 왼쪽 클릭)하면 링크가 새 탭으로 열릴 것입니다. 하지만 새로 열린 탭이 바로 보이지는 않고, 배면 탭으로 열릴 것입니다. 이 &pref.singular;을 체크하면 링크가 전면 탭으로 열려 바로 보일 것입니다. (즉, 포커스가 새 탭으로 바로 전화되어 새 탭이 현재 탭이 됩니다.)

&brandShortName;는 속도, 보안, 접근성, 또는 개인 취향 등 다양한 이유로, 사용자가 원할 경우, 특정 웹 내용을 차단할 수 있도록 합니다.

팝업 창 차단
기본 적으로 &brandShortName;는 웹 사이트에 나타나는 귀찮은 팝업 창을 차단합니다. 이 &pref.singular;을 선택 해제하면 팝업 차단 기능을 사용하지 않게 됩니다.

일부 사이트는 정당한 목적으로 팝업 창을 사용합니다. 따라서, 이런 사이트에 대해서는 팝업을 허용해야 합니다. 이렇게 팝업을 허용하려면, 예외 목록을 클릭하여 사이트 이름을 입력하고 허가을 클릭합니다. 목록에서 사이트를 제거하려면, 해당 사이트를 선택하고 제거를 클릭합니다. 목록 전체를 초기화 하려면, 모두 제거를 클릭합니다.

그림 자동으로 읽기
기본적으로, 웹 페이지에 있는 그림을 읽어 옵니다. 이 &pref.singular;을 선택 해제하면 그림을 전혀 불러오지 않을 것입니다.

어떤 사이트를 차단하고 어떤 사이트를 그림을 불러오도록 허용할지를 변경하려면, 예외 목록을 클릭합니다. 그림을 허용/거부할 사이트를 입력하고 허가를 클릭하여 그림을 허용하든지, 아니면 차단을 클릭하여 이미지를 차단합니다.

파일 형식

다운로드 실행 방식 대화 상자는 파일 형식… 단추를 클릭하면 열리며, 사용자가 다운로드 하는 파일 형식 목록을 담고 있습니다. 특정 파일 형식을 클릭하면 &brandShortName;가 실행할 일은 파일 형식을 선택하고 편집… 단추를 클릭하여 지정할 수 있습니다.

실행 방식 대화 상자에서 파일 형식을 기본 응용 프로그램으로 열 것인지, 사용자 지정 응용 프로그램으로 열 것인지, 디스크에 저장할 것인지, 아니면 설치된 플러그인을 사용하여 보여줄 것인지를 선택할 수 있습니다. 예를 들어, 웹 페이지에서 많은 수의 미디어 파일을 본다면, &brandShortName;가 매번 미디어 파일을 어디에 저장할지를 물어오는 것보다는 항상 미디어 재생기를 사용하여 열기를 선호할 것입니다.

  • 기본 응용 프로그램으로 열기:
    이 &pref.singular;을 선택하면 (OS가 인식한) 파일 형식을 기본 응용 프로그램으로 엽니다.
  • 다음 용용 프로그램으로 열기:
    이 &pref.singular;을 선택하여 이 파일 형식을 처리할 다른 응용 프로그램을 지정합니다. 대화 상자에서 사용할 응용 프로그램을 지정하라고 물어올 것입니다.
  • 내 컴퓨터에 저장하기:
    이 &pref.singular;을 선택하면 파일을 디스크에 저장할 것입니다. 일반 설정에서 다운로드 저장 위치를 지정하였다면, 파일이 자동적으로 이 폴더에 저장될 것입니다.
  • 다음 플러그인 사용하기:
    이 &pref.singular;을 선택하면 플러그인이 해당 파일 형식을 처리하게 됩니다.

파일 형식에 대한 자동 규칙을 제거하려면, 다운로드 실행 방식 대화 상자에서 해당 파일 형식을 선택하고 제거 단추를 클릭합니다.

쿠키

쿠키는 웹 사이트가 생성한 파일로 방문시 사용자의 컴퓨터에 사용자 정보를 저장해 놓을 것입니다.

웹 사이트에서 제공하는 쿠키 허용
기본적으로 쿠키는 사용 가능하게 되어 있습니다. 이 &pref.singular;을 선택 해제하면 쿠키 사용이 불가능하게 됩니다. 일부 사이트는 쿠키 사용이 정지되면 정상적으로 작동하지 않을 수도 있음을 유념합니다.

유지 기간:

  • 만료 될 때까지: 이 &pref.singular;을 선택하면, 각 쿠키는 만료 날짜가 되면 삭제될 것입니다.
  • &brandShortName; 닫을 때까지: 이 &pref.singular;을 선택하면, 쿠키는 &brandShortName;를 재시작할 때 삭제될 것입니다. 쿠키 사용을 요구하는 웹 사이트는 문제 없이 작동은 할 것이지만, 이 웹 사이트는 사용자가 처음으로 접속하는 것으로 인식할 것입니다.
  • 매번 물어 보기: 웹 사이트가 쿠키 저장을 시도할 때마다 대화 상자가 나타나서 해당 쿠키를 저장할 것인지를 물어올 것입니다.

쿠키 메니저를 보려면 쿠키 보기를 클릭합니다.

쿠키와 쿠키의 각 &pref.plural;, 그리고 어떤 사이트가 사용자 컴퓨터에 쿠키 저장할 수 있도록 허용할 것인지를 제어하는 방법에 대한 더 자세한 정보는 쿠키 관리하기를 참조합니다.

프록시 수정 설정
프록시를 자동 설정해 주는 주소(URL)를 이 &pref.singular;을 선택합니다. 시스템 관리자에게 각 네트워크 서비스에 대해 프록시 소프트웨어를 실행하는 서버의 이름과 포트 번호를 문의하여 이 정보를 알맞은 필드에 입력합니다.
프록시 자동 설정 주소 (URL)
사용자의 작업 장소에 대한 프록시 설정 파일이 있으면, 시스템 관리자에게 URL을 문의하여 여기에 입력합니다. 새로 고침을 클릭하여 설정 변경 사항을 다시 불러옵니다.

캐시

사용자가 열어 본 페이지는 보통 캐시라는 특별한 폴더에 저장되어, 나중에 동일한 페이지를 방문할 때 빠를게 보여 줍니다. 여기에 캐시가 사용할 디스크 공간을 지정할 수 있습니다.

사용 용량
사용자 컴퓨터에서 사용할 캐시의 최대 크기를 메가바이트 단위로 지정합니다.

지금 삭제
즉시 현재의 캐시 내용을 삭제하여, 캐시에 의해 사용되던 디스크 공간을 비웁니다.

업데이트 탭

&brandShortName;는 설치된 확장 기능, 테마, 또는 &brandShortName; 자체에 대해 업데이트할 것이 있는지를 확인하게 할 수 있습니다.

자동으로 업데이트 할 항목:
기본적으로 &brandShortName;는 자동으로 자기 자신과, 부가 기능(확장 기능 및 테마), 그리고 검색 엔진에 대한 업데이트 확인하여 사용자가 최신 버전을 가지고 있는지 알려줍니다. 이 동작을 사용하지 않으려면, 해당 확인 상자를 선택 해제합니다.

&brandShortName; 업데이트 항목이 있는 경우:

항상 물어 보기
&brandShortName;에 대한 업데이트는 보통 자동적으로 설치됩니다. 이 &pref.singular;을 선택하면, 어떻게 언제 업데이트를 설치할 것인지 수종으로 제어할 수 있습니다.
자동으로 다운로드하여 설치하기
이 &pref.singular;을 선택하면, &brandShortName; 업데이트가 자동적으로 다운로드되어 &brandShortName;를 다음에 재시작할 때 설치될 것입니다. Th부가 기능을 사용할 수 없는 경우 알림 &pref.singular;은 새로운 호환 버전이 없어 사용이 불가능할 수 있는 비호환 확장 기능이나 테마에 대한 업데이트를 설치하기 전에 경고 메시지를 표시할지 말지를 지정합니다. 경고 메시지는, 업데이트에 포함된 성능향상에도 불구하고, 업데이트 설치를 연기할 것인지를 물을 것입니다.
모든 확장 기능과 테마가 호환성 있거나 업데이트 되어 호환성 있게되는 경우, &brandShortName; 업데이트가 설치될 것입니다. 재시작 시 요구되는 확장 기능과 테마 업데이트를 설치하여 사용자가 계속해서 해당 확장 기능과 테마를 사용할 것인지를 물어올 것입니다.

업데이트 기록 보기
&brandShortName;는 자동적으로 사용자가 설치한 업데이트를 기록합니다. 업데이트 기록 정보를 보려면 이 단추를 클릭합니다.

주의: 관리자 루트나 처음 &brandShortName;를 설치한 사용자로 &brandShortName;를 실행하고 있어야만 &brandShortName; 업데이트를 설치할 수 있습니다.

암호화 탭

프로토콜

SSL 3.0 사용
보안 정보를 SSL3(Secure Sockets Layer, Level 3)를 통해 송수신할 것인지를 지정합니다. 이것은 웹 사이트와 안전하게 통신하기 위한 표준 프로토콜입니다. 이 프로토콜을 사용 정지시키면 일부 보안 사이트를 방문할 수 없게 될 것입니다.

TLS 1.0 사용
보안 정보를 SSL3(Secure Sockets Layer)과 비슷한 오픈 보안 표준인 TLS(Transport Layer Security)을 통해 송수신할 것인지를 지정합니다. 이 프로토콜을 사용 정지시키면 일부 보안 사이트를 방문할 수 없게 될 것입니다.

인증서

인증서는 보안 사이트에 대한 접속을 암호화하고 복호화하는데 필요합니다.

웹 사이트에서 인증서를 요구할 경우:
자동으로 한가지 선택를 선택하면, 웹 사이트가 보안 접속을 요청하면, &brandShortName;는 기본적으로 자동으로 적당한 인증서를 이용합니다. 수동으로 인증서를 선택하려면 (예를 들어, 자동으로 선택된 것 대신 특정 형식의 암호화를 선택하려면), 항상 물어 보기 &pref.singular;을 선택하면 탐색하는 동안 어떤 인증서를 사용할 지를 완전히 제어할 수 있게될 것입니다.

인증서 보기
이 단추를 클릭하여 저장된 인증서를 보거나, 새로운 인증서를 불러오고 저장하거나 또는 &brandShortName;에 있는 오래된 인증서를 삭제할 수 있습니다.

인증서 폐기 목록
&brandShortName;는 (CRL이라고 하는) 인증 폐기 목록을 사용하여 사용자의 인증서가 무효하지 않음을 보증합니다. 인증서 폐기 목록 단추를 클릭하면 사용자 컴퓨터에 설치되어 있는 CRL을 관리할 수 있습니다.

유효성 확인
OCSP(온라인 인증서 상태 프로토콜)는 인증서가 표시되거나 사용될 때마다 유효성을 검사하는 방법입니다. &brandShortName;는 기본적으로 OCSP를 사용하지 않지만, 여기서 사용가능하게 지정할 수 있습니다. 인터넷 환경이 이를 요구할 때에만 이 설정을 변경시킬 필요가 있을 것입니다.

보안 장치
보안 장치는 접속을 암호화하고 복호화할 수 있으며 인증서와 암호를 저장할 수 있습니다. &brandShortName;에 있는 것 이외의 보안 장치를 사용하려면, 보안 장치 단추를 누릅니다.
